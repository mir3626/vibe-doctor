# v1.1.0 Harness Improvements — run-codex sandbox filter + sprint-complete LOC + preflight orchestration check

## 배경

vibe-doctor v1.1.0 은 주로 문서(CLAUDE.md, SKILL.md, _common-rules.md, 신규 orchestration.md)
수정을 통한 프로세스 재정렬이다. 남은 코드 변경 3개를 이 Sprint가 담당한다.

`.vibe/agent/_common-rules.md` 공용 규칙 준수. 샌드박스 우회 금지. 의존성 설치 금지.

## 목표

1. Codex Generator의 Final report에서 "Sandbox-only failures" 섹션이 유의미하게 (≥1 항목) 존재할 때, run-codex.sh가 실행 결과 맨 아래에 **한 줄 요약 + 필터링된 블록**을 출력하여 Orchestrator가 sandbox-only vs real failure를 즉각 구분할 수 있게 한다.
2. `vibe-sprint-complete.mjs` 가 Sprint 완료 판정 시 **git diff 기반 실측 LOC**를 측정하여 sprint-status.json 의 해당 sprint entry에 `actualLoc` 필드로 기록한다.
3. `vibe-preflight.mjs` 가 `docs/context/orchestration.md` 파일 존재를 **정보성 체크**로 확인한다 (미존재 시 경고만, FAIL 아님).

## 수정할 파일

### 1. `scripts/run-codex.sh` (POSIX shell)

Codex exec 출력의 마지막 처리 부분(기존 retry/pipe 로직 끝 부분)에, Codex가 반환한 Final report 텍스트에서 `## Sandbox-only failures` 섹션을 파싱하여 다음을 stderr에 덧붙여 출력:

```
──── run-codex.sh: Sandbox-only failures 감지 ────
(N개 항목. Orchestrator가 샌드박스 밖에서 재검증 필요.)
<해당 섹션 body 그대로, 최대 20 lines>
───────────────────────────────────────────────
```

- `## Sandbox-only failures` 헤더가 없거나 그 아래 body가 `- none`/비어있으면 출력 생략
- Codex 정상 출력은 그대로 보존 (기존 stdout 스트리밍 비간섭)
- bash + POSIX sed/awk/grep 조합으로 구현. Windows Git Bash에서도 돌아야 함

구현 힌트: Codex stdout을 임시 파일에 `tee` 해두고, 종료 후 해당 섹션을 grep/awk로 추출. 또는 라인 buffer에서 "## Sandbox-only failures" 이후부터 다음 "## " 또는 EOF까지를 추출.

### 2. `scripts/vibe-sprint-complete.mjs` (Node ESM)

현재 이 스크립트는 sprint-status.json 갱신 + handoff.md + session-log.md append 를 수행. 여기에 **git diff 기반 LOC 측정** 을 추가:

- 인자: `<sprintId> <passed|failed>` (기존 유지)
- 추가: 직전 커밋(HEAD) 과 그 전 커밋(HEAD~1) 사이 diff로 LOC 계산. 또는 현재 working tree + staged 변경 기준으로 삽입(insertions)/삭제(deletions) 집계.
  - `git diff --shortstat HEAD~1 HEAD` 또는 `git diff --shortstat --cached` 사용
  - 단, Sprint 최초 1회(HEAD~1이 존재하지 않는 경우)는 `git diff --shortstat HEAD` + 초기 커밋 처리
- sprint-status.json 의 `sprints[]` 에서 해당 sprintId entry에 다음 필드 append:
  ```json
  {
    "id": "sprint-NN-...",
    "status": "passed",
    "completedAt": "...",
    "actualLoc": {
      "added": 123,
      "deleted": 45,
      "net": 78,
      "filesChanged": 6
    }
  }
  ```
- git 명령 실패 시 `actualLoc` 필드 생략(=기록 안 함). 전체 스크립트는 계속 진행.
- session-log.md append 메시지에 `LOC +123/-45 (net +78)` 한 줄 포함.

### 3. `scripts/vibe-preflight.mjs` (Node ESM)

기존 체크 목록 끝에 `orchestration.md` 존재 체크 추가. 코드 대략:

```js
const orchestrationPath = resolve('docs/context/orchestration.md');
if (existsSync(orchestrationPath)) {
  record('orchestration.doc', true, 'present');
} else {
  record('orchestration.doc', true, 'missing (optional shard — v1.1.0+)');
}
```

- 미존재는 FAIL이 아님 (경고만). 구버전 프로젝트가 아직 sync하지 않았을 수 있으므로.
- record() 헬퍼는 기존에 있음. `true` 로 마킹하고 detail 메시지에 상태 기술.
- bootstrap 모드에서도 동일하게 동작 (skip 분기 불필요).
