# Sprint 실패 에스컬레이션

## 조건

- 같은 Sprint에서 Evaluator가 2회 연속 불합격 판정

## 절차

1. Orchestrator가 불합격 사유를 분석
2. 패턴 판별:
   - **스펙 문제** → Planner 재생성하여 체크리스트 수정
   - **구현 문제** → Generator에게 구체적 수정 지시 전달
3. 수정된 입력으로 Generator 재생성
4. Evaluator 재판정
5. 3회 연속 불합격 시 사용자에게 에스컬레이션 — 선택지 제시:
   - 스펙 범위 축소
   - 기술 스택 변경
   - 수동 개입
6. 최종 결과와 에스컬레이션 사유를 보고서에 기록
