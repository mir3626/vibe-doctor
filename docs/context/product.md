# Product context

이 저장소는 **SpotDiff** — 사용자가 업로드한 이미지를 AI가 변형하여 틀린그림찾기 게임을 즐길 수 있는 모바일 웹앱이다.

## 성공 기준
- 사용자가 기기 갤러리에서 이미지를 선택·업로드할 수 있다
- AI(Gemini 2.5 Flash Image)가 원본 이미지에서 6곳을 자연스럽게 변형한다
- 좌(원본)·우(변형본) 배치로 30초 제한 틀린그림찾기 게임을 플레이할 수 있다

## 플랫폼
- 모바일 웹 전용 (데스크톱 미지원)

## 핵심 기능 흐름

### 1. 이미지 업로드
- 기기 갤러리에서 이미지 선택
- 클라이언트 전처리: 최대 1024x1024~2048x2048 리사이즈, 2~3MB 이하 압축 후 서버 전송

### 2. AI 변형 이미지 생성
- 원본 이미지 + 상세 프리셋 프롬프트 → Gemini API 호출 → 6곳 변경(요소 변경/추가/제거)
- 백엔드: Gemini API 프록시 + 픽셀 diff 처리
- 정답 검출: 픽셀 diff 자동 검출(주) + AI 프롬프트 좌표 출력(보조)
- diff 처리 위치(서버/클라이언트)는 Generator 재량

### 3. 대기 화면
- 재미있는 애니메이션 + 메시지 ("AI가 그림을 바꾸는 중..." 등)
- API 응답에 예상 완료시간 포함 시 로딩바 + 진행 메시지

### 4. 게임 플레이
- 배치: 좌(원본) / 우(변형본)
- 틀린 부분: 6개
- 시간 제한: 30초
- 오답 탭: 3초 프리징 (입력 불가)
- 클리어: 6개 전부 찾기
- 게임오버: 시간 초과

### 5. 결과 화면
- 결과 요약 + 못 찾은 부분 하이라이트
- "같은 이미지로 재도전" / "새 이미지로 시작" 버튼
- SNS 공유: 결과 스크린샷 이미지를 클라이언트에서 생성하여 공유

### 6. 데이터 저장
- 변형 이미지, 정답 좌표, 게임 결과 → 브라우저 로컬스토리지 (서버 측 저장 없음)

## AI 이미지 생성
- 모델: `gemini-2.5-flash-image` (Nano Banana)
- 변형 방식: 원본 이미지 + 상세 프리셋 프롬프트 → 일부 요소 변경/추가/제거
- 모델은 이미지 품질에 따라 추후 변경 가능

## UI/UX 요구사항
- AI-slop 디자인 지양 (그라디언트 남발, 둥근 카드 나열, 무의미한 일러스트 금지)
- 개성 있고 직관적인 인터페이스
- 게임 특성에 맞는 몰입감 있는 디자인
- 터치 인터랙션에 최적화된 UX

## 핵심 가정 (ouroboros 인터뷰에서 드러난 사항)
- 프로토타입에서는 사용자가 직접 이미지를 업로드하는 방식 (앱 내 앨범 없음)
- AI가 변형한 위치 정보는 픽셀 diff로 자동 검출 (AI 좌표 출력은 보조)
- 큰 이미지는 클라이언트 전처리로 압축 후 전송
- 게임 데이터는 서버 저장 없이 로컬스토리지만 사용
