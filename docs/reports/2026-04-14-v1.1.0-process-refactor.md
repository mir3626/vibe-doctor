# v1.1.0 — Process refactor: 하네싱 경계 재정렬

**Date**: 2026-04-14
**Scope**: dogfood5 회고에서 발견된 두 가지 하네싱 위반(Ouroboros 인터뷰 스킵 / Planner 소환 단위 오독)을 템플릿 자체에서 근본 수정 + Troubleshooting 개선 코드 적용
**Version**: `harnessVersion` 1.0.3 → **1.1.0** (minor — 프로세스 변경)

## 왜

dogfood5에서 다음 두 가지 하네싱 오독이 발생했다:

1. **Phase 3 Ouroboros 인터뷰 스킵** — 사용자의 "자율 진행" 지시를 "인터뷰 flow 자체 생략 가능" 으로 오해. 실제 의도는 "사용자 대신 답변하며 flow 완주".
2. **Planner 소환 단위 오독** — "새 프로젝트 첫 Sprint 분할" Must 트리거를 "Planner 1회 호출로 전체 로드맵 + Sprint 1" 까지 산출하는 것으로 해석. 실제는 "매 Sprint 시작 전 Planner 소환 + 로드맵은 Orchestrator 책임".

근본 원인은 **템플릿 문구의 모호성**. 다음 dogfood 라운드 전에 템플릿 자체를 명확화하지 않으면 같은 오독이 재발할 위험이 있어 v1.1.0 minor bump로 재정렬.

상세 분석 + 개선 제안: `docs/plans/dogfood6-improvements.md`.
