# v1.8.5 - Pro-Go Publish Confirmation Skip Directive

Date: 2026-07-20

## Changes

- Added `vibe-pro-go confirm-skip on|off|status`. `on` records the
  `userDirectives.proGoAutoPublish` directive in `.vibe/config.local.json`
  (optional `--reason`, optional `--days 1-365` expiry) and appends a
  session-log `[decision][pro-go-auto-publish]` entry; `off` disables it with a
  `[decision][pro-go-auto-publish-clear]` entry; `status` reports the effective
  state.
- `go` and `status` output now include `autoPublish`. While it is `true`, the
  `$vibe-pro-go` runbook skips the per-write user authorization wait: the agent
  still shows repository, branch, target, and files, then passes `--publish`
  and records one `[decision][auto-approved]` session-log entry per
  publication.
- The CLI `--publish` sentinel contract is unchanged; the directive only
  governs the agent-side confirmation stop. A missing, malformed, or expired
  directive always falls back to requiring explicit authorization.

## Compatibility

- No Pro protocol, schema, flow archive, default-branch, or PR policy changed.
- `.vibe/config.local.json` remains gitignored and user-local; downstream repos
  opt in per checkout.
- Default behavior without the directive is identical to v1.8.4.
