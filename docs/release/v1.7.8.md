# v1.7.8 - Review Parser and Dogfood Follow-Up Hardening

Date: 2026-05-07

LTS: `v1.7.3-lts` remains the stable baseline. This is a mainline `/vibe-review` hardening release from `codex-widget-for-desktop` dogfood.

## Changes

- Replaced direct harness-gap ledger `.split('|')` parsing with a Markdown table splitter that respects escaped pipes and inline code spans.
- Added regression rows for covered ledger entries containing:
  - `bundle.policy=automatic|custom|off`
  - shell examples such as `\|\| true`
  - archive matching expressions such as `base === sprintId \|\| base.startsWith(...)`
- Relaxed prior-review parsing to accept the documented `## Findings (severity desc)` heading in addition to exact `## Findings`.
- Normalized legacy minimal `.vibe/agent/project-decisions.jsonl` records with `timestamp`, `tag`, `decision`, and `reason` instead of warning on valid older bootstrap records.
- Updated `/vibe-review` guidance to keep context-audit signals compact and report-only.
- Expanded semantic acceptance guidance so agent-context, multimodal, and workflow-agent features need dogfood transcripts, screen-share notes, task-quality artifacts, or before/after task outcomes beyond type/build/smoke checks.

## Non-goals

- No prompt/capsule/context reduction work.
- No context-audit preflight or sprint-commit gate.
- No downstream product-code changes.
- No semantic rewrite of legacy decision records back to disk; normalization happens at read time.
