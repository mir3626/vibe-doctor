# v1.7.20 - Downstream-Safe Template Hygiene Test Gating

Date: 2026-05-29

## Changes

- The `template project-owned hygiene` self-test now runs its pristine-template
  assertions (empty `docs/prompts`/`reports`/`archive`, not-initialized context
  shards, empty runtime state) only when the repo is the uninitialized upstream
  template.
- Template detection keys off the `PROJECT NOT INITIALIZED` marker in
  `.vibe/agent/handoff.md`, which stays authoritative even though `project.name`
  remains `vibe-doctor` downstream.
- The dashboard/report render-template structure check stays a generic
  harness-integrity test and continues to run in every project.

## Compatibility

- Initialized downstream projects no longer fail `npm run vibe:self-test` on the
  template hygiene suite; those assertions skip by design when real project
  docs/state are present.
- The upstream template repo behavior is unchanged: it is pristine, so all
  assertions still execute and gate template hygiene.

## Verification

- `node --import tsx --test .vibe/harness/test/template-hygiene.test.ts`
- `npm run vibe:typecheck`
- `npm run vibe:self-test`
