# v1.7.11 - Sidecar Dogfood Hardening

Date: 2026-05-08

LTS: `v1.7.3-lts` remains the stable baseline. This is a mainline hardening patch for the manual `diff-reviewer` sidecar foundation.

## Changes

- Fixed Windows/Codex sidecar execution by resolving the npm-installed `@openai/codex/bin/codex.js` entrypoint and invoking it through `process.execPath`.
- Kept non-Windows Codex execution on the direct `codex` command path.
- Recomputed and verified `inputHash` when `--input-file` is supplied, rejecting mismatches before artifact creation.
- Documented that saved or hand-edited `--input-file` packets must be regenerated when the packet body changes.
- Made artifact coverage wrapper-owned from the sealed packet; reviewer stdout can no longer clear `coverage.truncated` or change file/diff counters.
- Added semantic reviewer-output validation:
  - `pass` requires zero findings.
  - `advisory` requires low/medium findings and rejects high findings.
  - `fail` requires at least one high finding.
- Resolved relative `--prompt-file`, `--input-file`, and `--mock-output-file` values against `--cwd`.
- Rejected `--artifact-root` values that resolve outside `--cwd`.
- Redacted/omitted diff content for sensitive tracked paths and credential-like extensions.
- Omitted untracked file contents by default, with `--include-untracked-content` available only for explicit local debugging.
- Omitted non-text or unsafe-control untracked content even when `--include-untracked-content` is explicitly enabled.
- Emitted neutral omission markers for redacted content instead of synthetic "new file" patches that could misrepresent the change type.
- Removed file-reading tools from the Claude `diff-reviewer` adapter metadata.

## Why

Downstream dogfood on `codex-widget-for-desktop` showed that vanilla v1.7.9 selected the Codex provider correctly but failed on Windows with `spawnSync codex ENOENT`. The root cause was Node process spawning not resolving the Windows npm shim even though PowerShell could run `codex`.

The same dogfood run also exposed sidecar wrapper contract risks: stale or tampered sealed packets, reviewer-owned coverage, contradictory status/findings combinations, relative path drift under `--cwd`, artifact-root escape, and accidental secret or untracked content exposure.
