# v1.8.1 — Web Pro Bridge 실전화 (remediation + MCP write-path)

v1.8.0 web-pro-bridge에 대한 **실 웹 ChatGPT Pro 리뷰**(AUD-20260715-tlo6jc, 11 findings)의 remediation과, 실 왕복에서 드러난 write-path 문제에 대한 후속 설계 패키지(`vibe-doctor-mcp-write-improvement-v1.8.0/`, MCP-001~007) 구현을 합친 릴리즈. Sprint 6개(vpb-07~12), per-sprint Evaluator 6회, 독립 whole-workflow 감사 2회(P0/P1 0) 통과.

## Remediation (iter-2 — 리뷰 P1×5 · P2×4 · P3×2 전항 코드 폐쇄)

- **Repository identity fail-closed** (P1-004): 현재 origin == request == result-manifest == provenance 4중 일치를 sync 전 경로·no-op·ack에 강제. 우회는 고마찰 플래그 + provenance 기록 + release 증거 제외
- **Mailbox 직렬화/fencing** (P1-001): per-request FIFO + 파일 lease + durable op 직전 fencing, stale owner 차단, pid+nonce 임시명
- **Finalize journal** (P1-002): 5-phase journal + 6-state 재시작 수렴, exact-manifest replay = 단일 revision
- **Install/ack 복구** (P1-003): receipt repo 바인딩, provenance 검증 후 멱등 ack, `--latest` 점유 방지
- **시맨틱 계약** (P2-003): FINDINGS.json v1 스키마 + manifest 카운트 대조 + 구현 프롬프트 9요소 검증
- **토큰 강화** (P2-004): one-time connect code → 인스턴스 세션 교환, `?token=` 제거, shutdown revoke
- **revN 일반화** (P3-002), **health/quarantine** (P3-001), **unbound 게이트** (P2-002), **App Server unavailable 확정** (P2-001)

## MCP write-path 개선 (iter-3 — MCP-001~007)

- **`publish_review_package`** (14번째 툴): 정상 패키지 1콜 발행 — 기존 검증 단일 경로 재사용, requestId/clientPublicationId/manifestSha 3중 멱등, 부분 result-ready 0. 한도 초과 시 `chunked-upload-required` fallback 계약(저수준 툴은 fallback 전용으로 격하)
- **완료 계약**: `get_request`에 publicationRequired/primaryFinalTool/chatOnlyOutputCompletesRequest=false — "Bridge가 result-ready를 반환하기 전까지 미완료" 프롬프트 규약
- **도구 메타데이터**: 14툴 전부 annotations/outputSchema/visibility/requiredScopes + 결정적 카탈로그 audit(`catalog-audit`, 스냅샷 회귀) + `bridge_capabilities`
- **doctor**: `vibe-pro-bridge.mjs doctor <url>` — 원시 tools/list 기반 7클래스 진단 (hook/QA 무결합)
- **auth 프로파일**: noauth-local(기본, 바이트 호환) / oauth(scope 5종 per-tool enforcement, insufficient_scope는 `_meta["mcp/www_authenticate"]` challenge, introspection 포트 주입형)
- **golden 데이터셋**: 5범주 15케이스 + 카탈로그 정합 자동 검증 + Developer Mode 수동 replay 절차 문서
- **연결 영속화**: `persistentCode`(config.local 옵트인, 재시작 생존 — 라이브 실측) + `tunnelUrl`(ngrok 고정 도메인) + `--rotate-code` → **ChatGPT 앱 1회 등록 후 영구 재사용**
