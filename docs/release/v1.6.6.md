# v1.6.6

## Summary

Adds app-code LOC threshold detection to the lightweight audit path so prototype Evaluator exemptions cannot silently survive after a project crosses the LOC budget.

## Changes

- Extended `scripts/vibe-audit-lightweight.mjs` to count app-code LOC under `audit.projectRoots` and default to `["src"]`.
- Added `audit.prototypeLocThreshold` support with a default of `2000`.
- Injects a `LOC_THRESHOLD_BREACH` pendingRisk with `level: "INFO"` when app-code LOC exceeds the threshold.
- Allows `.vibe/config.local.json` to override audit roots/threshold for local harness checkouts without changing synced project config.
- Updated the Evaluator trigger matrix so an open `LOC_THRESHOLD_BREACH` invalidates the prototype exception.
- Documented the gap closure in `docs/context/harness-gaps.md`.
