# v1.8.2 - GitHub-backed Web Pro Round Trip

Date: 2026-07-20

## Changes

- Added `$vibe-pro-go` and `npm run vibe:pro-go` as the minimal Codex entrypoint.
  A bare invocation selects the newest matching non-closed flow, synchronizes
  the durable packet from the configured GitHub bridge branch, restores the
  current Sprint envelope, and continues from the next valid action.
- Added root `bridge-runbook.md` for Web Pro-origin work. It supports review
  requests such as reviewing work since a commit or auditing a project for
  improvements, then creates the goal and detailed design directly through the
  GitHub connector without requiring a CLI-generated prompt.
- Required exact repository and non-default ref access in Web Pro. The runbook
  explicitly forbids Web Search, GitHub search-index, default-branch, and
  implicit fallback behavior for bridge reads and writes.
- Added the flat `flows/YYYYMMDD/NNN-slug/` append-only archive, immutable event
  directories, `FLOW.json`, durable contract packets, and completion markers.
- Added exact design, Sprint, and commit binding plus Sprint-local, cumulative,
  and final workflow gates. Active Pro-origin Sprints cannot pass completion
  with stale or incomplete cross-Sprint evidence.
- Wired the shared workflow-integrity rules into planning, goal iteration,
  Sprint iteration, and report generation to limit overengineering and reduce
  workflow desynchronization after context compaction.
- Added automatic Web-facing implementation and remediation report preparation
  for goals, iterations, or Sprints that started through `$vibe-pro-go`.

## Compatibility and Migration

- The dedicated bridge branch remains `origin/vibe-pro-bridge`; it is used as
  append-only shared storage and is not merged into the product implementation
  branch.
- The former MCP bridge implementation and its historical artifacts remain
  preserved on `origin/vibe-pro-bridge`.
- Migration `1.8.2` removes only legacy MCP bridge harness files that match a
  recorded sync hash or a known published v1.8.0-v1.8.1 lineage blob. Modified
  or unknown files are retained and listed in
  `.vibe/harness-migration-1.8.2.md`.
- Historical `.vibe/pro-bridge/**` results and project-owned design/review
  documents are intentionally preserved.
- The migration removes the exact former default `proBridge` configuration and
  exact legacy `vibe:pro-*` package commands. Customized configuration or
  commands are retained.
