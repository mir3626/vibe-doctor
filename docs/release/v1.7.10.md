# v1.7.10 - Offset Datetime Schema Compatibility

Date: 2026-05-08

LTS: `v1.7.3-lts` remains the stable baseline. This is a mainline CI compatibility patch for synced downstream projects.

## Changes

- Added a shared `IsoDateTimeSchema` that accepts ISO datetimes with explicit offsets.
- Updated harness state schemas to accept values such as `2026-05-08T02:25:00.000+09:00`:
  - `sprint-status.json`
  - `project-map.json`
  - `sprint-api-contracts.json`
  - `model-registry.json`
  - sidecar artifact metadata
- Added a regression test for offset timestamps in `sprint-status.json`.
- Regenerated checked-in JSON schemas.

## Why

Downstream GitHub CI on `codex-widget-for-desktop` failed after harness sync because Linux `npm test` parsed the production `.vibe/agent/sprint-status.json` and rejected local timezone timestamps written on a Korea/Windows workflow. The previous Zod default accepted `Z` timestamps but not explicit offsets.

## Verification

- `node --import tsx --test .vibe/harness/test/sprint-status.test.ts .vibe/harness/test/schemas.test.ts`
- `npm run vibe:gen-schemas -- --check`
- `npm run typecheck`
