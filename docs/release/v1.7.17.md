# v1.7.17 - Injection and Sharding Safety Hardening

Date: 2026-05-14

LTS: `v1.7.3-lts` remains the stable baseline. This is a mainline harness safety patch.

## Changes

- Adds Codex wrapper Markdown injection diagnostics with JSON output and no Codex execution side effects.
- Enables transitive injection of explicit shared skill shard blocks so compact Codex wrappers can still load required runbook shards.
- Splits `/vibe-init`, `/vibe-interview`, `/vibe-iterate`, and `/vibe-review` into guarded shards with dedicated audit scripts, CI wiring, and preflight checks.
- Adds an all-skill Codex wrapper audit for wrapper/shared skill inventory, unsafe path rejection, transitive shard targets, and compact skill injection targets.
- Adds sprint-mode and sync boundary audits, including guard coverage that prevents full harness ownership of root `README.md`.
- Splits dashboard and project-report render templates into `.vibe/harness/scripts/lib/*` modules while preserving synchronous render flow.
- Updates README, upgrade, troubleshooting, sync, and injection guarantee docs to match the current `.vibe/harness/**` runtime layout and script inventory.

## Why

The wrapper and skill runbooks now rely on sharded Markdown. Without diagnostic and audit coverage, a future edit could silently omit injected context or move sync ownership across downstream project boundaries. This release makes those contracts machine-checkable before sync, CI, and release.
