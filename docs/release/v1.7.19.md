# v1.7.19 - Review Wiring Shards and Sprint Contract Ledger

Date: 2026-05-19

## Changes

- `/vibe-review` wiring drift detection now follows explicit `*SHARDS`
  marker blocks in skill Markdown before deciding whether a harness script has
  a runtime reference.
- Fixed the self-reported `vibe-review-inputs.mjs` wiring drift signal caused
  by references that lived in sharded review protocol Markdown rather than the
  root `SKILL.md`.
- Added a required `Sprint Contract` block to Planner-generated Sprint prompts
  and Generator final reports. The block records target/output surface,
  allowed writes and exclusions, explicit exceptions, reference-only values,
  proof predicates, current proof, and non-proof.
- Added regression coverage for sharded wiring references and Sprint Contract
  proof-boundary fields.

## Compatibility

- Only explicit `*SHARDS` marker blocks are followed for wiring reference
  discovery; ordinary Markdown links remain non-recursive.
- Sprint prompts and Generator reports should now include the compact
  `Sprint Contract` section. Existing downstream projects can adopt it on their
  next generated Sprint prompt.
- Provider bridge verification and installer/update patterns remain deferred
  until there is a concrete host portability incident.
