# v1.4.0 (2026-04-16)

## M-audit

- Added Zod as the single runtime source for sprint-status, project-map, sprint-api-contracts, iteration-history, and model-registry state validation.
- Added `scripts/vibe-gen-schemas.mjs` to regenerate or check `.schema.json` files from the Zod source.
- Added `scripts/vibe-validate-state.ts` and wired preflight/sprint-complete state validation through the shared schema layer.
- Promoted audit cadence from documentation to a preflight gate with `--ack-audit-overdue=<sprintId>:<reason>` escape hatch and permanent session-log recording.
- Added `scripts/vibe-audit-lightweight.mjs` for non-blocking per-sprint diff/spec/test/tmp residue checks.
- Added migration `migrations/1.4.0.mjs` for bootstrap defaults: top-level `verificationCommands`, nullable `verifiedAt`, and handoff budget/preferences fields.

## Notes

- Zod-derived JSON schema output may omit legacy hand-written metadata such as `x-purpose`.
- Full production build and full test suite remain Orchestrator post-handoff responsibilities under the sandbox contract.

## M-process-discipline

- Renamed `.claude/agents/planner.md` → `.claude/agents/sprint-planner.md` to remove collision with Claude Code's built-in read-only Plan agent. New frontmatter explicitly declares `tools: Read, Glob, Grep, WebFetch, Write, Edit`.
- Extended `migrations/1.4.0.mjs` with file-level migration (`migrateAgentFiles`) — safely removes orphan `.claude/agents/planner.md` after downstream sync installs the new file.
- Added `scripts/vibe-planner-skip-log.mjs` CLI — enforces a recorded `[decision][planner-skip]` entry in session-log when the trivial exception is invoked (replacement for the previous informal skip).
- Added preflight `planner.presence` check — non-blocking WARN if the next pending sprint has no prompt file and no recorded skip decision.
- Revised CLAUDE.md "trivial" exception: LOC-based threshold replaced with a semantic three-condition test (pattern inheritance + no architecture change + ≤3 checklist items). Mirrored into `.vibe/agent/_common-rules.md` §10.

## M-harness-gates

- Added harnessVersion delta auto-tagging to `scripts/vibe-sprint-commit.mjs`; tags are annotated and remote push remains opt-in via `--push-tag`.
- Extended `docs/context/harness-gaps.md` with `script-gate` and `migration-deadline` columns, covering MD-only rule audit and release tag automation gaps.
- Added `scripts/vibe-rule-audit.mjs` to surface CLAUDE.md imperative rules that lack covered script-gate ledger entries.
- Added `scripts/vibe-audit-skip-set.mjs` plus `auditSkippedMode` local config shape for persistent, expiring user audit-skip directives.
- Updated preflight audit overdue handling to demote FAIL to WARN while an active audit-skip directive is still within `expiresAt`.
