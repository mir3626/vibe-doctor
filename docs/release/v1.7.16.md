# v1.7.16 - Sprint-Mode Statusline Polish

Date: 2026-05-11

LTS: `v1.7.3-lts` remains the stable baseline. This is a mainline statusline polish patch.

## Changes

- Hides the sprint-mode statusline segment when `/vibe-sprint-mode` is `off`.
- Shows active sprint-mode states with an emoji segment:
  - `🏃 sprint:core`
  - `🏃 sprint:extended`
  - `🏃 sprint:partial`
- Keeps the v1.7.15 wildcard permission behavior unchanged.

## Why

The `off` state is the default/inactive state and was adding visual noise. Active sprint-mode states are now easier to scan because they follow the same emoji-led convention as time, token, warning, and version segments.
