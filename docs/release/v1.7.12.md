# v1.7.12 - Provider-Neutral Orchestrator Wording

Date: 2026-05-08

LTS: `v1.7.3-lts` remains the stable baseline. This is a mainline documentation and synced agent-memory patch.

## Changes

- Updated `AGENTS.md` so Sprint Generator escalation goes to the current upstream Orchestrator instead of hard-coding a Claude-specific Orchestrator.
- Reworded `CLAUDE.md` to make Claude the nominal Orchestrator for Claude sessions without claiming Claude is the only valid Orchestrator.
- Reworded README overview, core design, and operating principles to distinguish:
  - default nominal mode: Claude Code Orchestrator + Codex Generator
  - direct Codex sessions: Codex Orchestrator maintenance mode for upstream harness work
- Updated `docs/context/orchestration.md` role table with the same distinction.

## Why

Downstream use can run `vibe-doctor` with Codex as the direct maintenance Orchestrator. Some synced docs still sounded like the harness strongly insisted that the Orchestrator must always be Claude. That is correct only for the nominal Claude Code operating mode, not for direct Codex maintenance sessions.
