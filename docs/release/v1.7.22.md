# v1.7.22 - Goal Iteration Skill and Scoped Verification

Date: 2026-06-10

## Changes

- Moved `vibe-goal-iterate` into the project-owned skill set with a shared
  runbook at `.claude/skills/vibe-goal-iterate/SKILL.md` and a Codex wrapper at
  `.codex/skills/vibe-goal-iterate/SKILL.md`.
- Added a `vibe-goal-iterate` verification override for multi-item loops:
  per-item cycles run only targeted checks tied to the current item.
- Reserved harness-wide or full-suite verification for the end of the final
  queued item, and only when the iteration explicitly changed
  harness/runtime/skill/orchestration behavior.
- Clarified that `npm run vibe:checkpoint` remains allowed during the loop
  because it checks durable context state rather than running broad
  verification.

## Compatibility

- Existing `/vibe-iterate` behavior is unchanged.
- Downstream projects can use `$vibe-goal-iterate` for long multi-item work
  without running unrelated harness-wide checks between items.
- Full harness verification remains appropriate for explicit harness changes
  at the end of an iteration or release boundary.

## Verification

- Shared and Codex `vibe-goal-iterate` skill validation with
  `quick_validate.py`
- `npm run vibe:codex-wrapper-audit`
- Markdown injection diagnostic for `.codex/skills/vibe-goal-iterate/SKILL.md`
- `npm run vibe:checkpoint`
- Encoding and mojibake checks for touched Markdown/YAML files
