# v1.10.0 - Finding-Scope Discipline for the Pro Review Loop

Date: 2026-07-22

Adopts the osint-stock-screener upstream proposal "general finding-scope
discipline for vibe-pro-go": after five consecutive evidence-plane P1
remediation rounds (the FND-019..024 series that became v1.9.0) stalled a
downstream product lane for three days, the review protocol gains a
domain-neutral scope discipline. The runbook supplies the mechanism; the
design event supplies the domain — the same pattern as `finalGatePolicy`.
The proposal's disposition-event protocol addition (implementer
`accept`/`defer`/`dispute` events) is deferred to the backlog pending real
need under the new discipline.

## Changes

- **Findings classification (schema, additive)**: `FINDINGS.json` findings
  accept optional `plane` (`product` | `evidence`), `impactClasses` (subset of
  six domain-neutral classes: `silent-incorrectness`, `overstated-validation`,
  `real-world-effect`, `irreproducibility`, `untrusted-boundary`,
  `unrecoverable-loss`), and `threatModel { actor, requiredCapability,
  productConsequence }`. New taxonomy `backlog-candidate` for
  real-but-non-blocking observations; a `backlog-candidate` can never be
  P0/P1 (schema-enforced).
- **Design-armed impact gate**: `CONTRACT.json` accepts an optional
  `productPlane { description, correctnessCritical, impactClasses,
  untrustedBoundaries }` block. When the active design declares it, the flow
  loader enforces that every P0/P1 finding claims at least one impact class
  and that every claimed class is design-declared
  (`validateFindingScopeDiscipline`). Flows without the declaration —
  historical flows and design-less audit flows — are exempt, so archived
  flows keep re-validating unchanged.
- **Evidence proportionality (publisher)**: `assertFinalEvidence` requires
  fresh `complete` workflow evidence only for contract rows a Sprint owns or
  affects (`owns` ∪ `workflowsAffected`); preserved rows are evidenced by the
  complete gate passing. An explicit non-complete row on a preserved
  invariant still blocks. The workflow matrix renders preserved-only rows as
  `preserved by cumulative gate` / `preserved` instead of `missing`.
- **Runbook (`WEB-RUNBOOK.md`)**: §4 documents the `productPlane`
  declaration; §5 gains the "Finding scope and severity discipline" section —
  plane classification, the impact gate, trust-boundary proportionality
  (no attestations for the process's own outputs), threat-model requirements
  for new invariants, self-reference (review machinery defects go to
  `harness-backlog` notes and upstream handoffs, never blocking findings),
  contract budget (no monotonic growth), evidence proportionality, the
  two-round cap per finding ID, backlog carry-forward into approval/close
  deferrals, and completion.
- **`docs/context/workflow-integrity.md` §8**: the CLI side applies the same
  discipline when self-reviewing and when deciding what to implement.

## Compatibility

- All schema additions are optional; historical `FINDINGS.json` and
  `CONTRACT.json` documents keep validating, and enforcement arms only when a
  design declares `productPlane`.
- Evidence-plane findings that pass the impact gate (e.g. a manifest forgery
  that overstates validation) can still be P0/P1 and still block — the
  discipline constrains what may block approval, not what may be found.
- The publisher change is a relaxation only; flows that recorded complete
  evidence for every row continue to pass unchanged. The matrix rendering of
  preserved-only rows changes from `missing` to `preserved`.
- The approval gate's blocking rule (unresolved P0/P1 of
  implementation-defect / missing-test / evidence-missing) is unchanged;
  `backlog-candidate` is non-blocking by construction.
