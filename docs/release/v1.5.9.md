# v1.5.9

## Summary

Fixes Claude Code statusline and hook compatibility when Claude is launched from Windows CMD or PowerShell.

## Changes

- Added `.claude/statusline.mjs` as the cross-platform statusline implementation.
- Changed `.claude/settings.json` to invoke `node .claude/statusline.mjs` instead of POSIX inline environment syntax.
- Removed POSIX-only `2>/dev/null || true` from the `SessionStart` hook command.
- Kept `.claude/statusline.sh` and `.claude/statusline.ps1` as compatibility wrappers around the Node implementation.
- Added regression tests that reject POSIX-only statusline/hook command wiring in Claude settings.

## Verification

- `npm run typecheck`
- `npm run build`
- `npm test`
