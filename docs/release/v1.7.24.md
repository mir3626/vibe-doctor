# v1.7.24 - Doc-Integrity Checkpoint Guard

Date: 2026-07-06

## Changes

- Added a `docs.integrity` check to `vibe-checkpoint.mjs`. Background: in a
  downstream project a `/maintain-context` pass truncated the project
  `CLAUDE.md` (357 lines) to 0 bytes via a shell one-liner that opened the
  file for writing while reading it in the same expression — and the
  checkpoint still passed green. The checkpoint now fails when a first-read
  boot document has been emptied.
- The check targets the fixed candidates `CLAUDE.md`, `AGENTS.md`,
  `GEMINI.md`, `.vibe/agent/_common-rules.md` plus the `docs/context/*.md`
  shards, restricted to files that are git-tracked and present on disk. A
  target fails when its content trims to empty or falls below a 64-byte
  threshold (`VIBE_DOC_MIN_BYTES` override).
- `maintain-context` SKILL.md (shared runbook + Codex wrapper note) gained
  two workflow rules: existing documents are rewritten with native
  Edit/Write tools only (no shell heredocs, no same-file
  read-while-write-open expressions), and `git diff --stat` is inspected
  before committing so an unintended mass deletion of a first-read document
  stops the pass.

## Compatibility

- Untracked or absent candidates (no `GEMINI.md`, partial or uninitialized
  checkouts) are silently skipped — the check targets "an existing document
  that got emptied", never "a document that was never there". When git is
  unavailable the check degrades to a best-effort size scan of files that
  exist on disk.
- `--json` output changes only additively (one new record entry).
- `--auto-refresh` never writes or "fixes" any of these documents — the
  check is report-only in both modes, because the correct response to an
  emptied document is restoration from git, not regeneration. The
  auto-refresh write path (handoff auto-state block) is unchanged from
  v1.7.23.

## Verification

- `npm run vibe:typecheck`
- `npm run vibe:self-test` (full harness suite: 467 pass / 0 fail / 1 skip)
- New `checkpoint.test.ts` coverage: healthy tracked docs pass, 0-byte
  tracked `CLAUDE.md` fails the checkpoint, missing/untracked `GEMINI.md`
  raises no false positive, whitespace-only `docs/context` shard fails.
