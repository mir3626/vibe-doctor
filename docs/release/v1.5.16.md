# v1.5.16

## Summary

Blocks `/vibe-sync` before `/vibe-init` has created project state, and resets copied template sprint history during agent-mediated initialization.

## Changes

- `/vibe-sync` now requires initialized project artifacts before planning or applying sync.
- `scripts/vibe-sync-bootstrap.mjs` applies the same pre-init guard for legacy one-shot bootstrap flows.
- `/vibe-init` agent bootstrap now rewrites copied template `.vibe/agent` state to an empty project state when the status file still belongs to `vibe-doctor`.
- Empty initialized state starts with `sprints: []` and `sprintsSinceLastAudit: 0`, so downstream projects no longer appear to have completed template sprints.
- Added regression coverage for sync-before-init, bootstrap-before-init, and template sprint state reset.
