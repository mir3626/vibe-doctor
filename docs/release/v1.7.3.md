# v1.7.3 - Review Policy Lifecycle Hardening

Date: 2026-05-04

LTS: `v1.7.3-lts` is an immutable alias for this release commit.

## Changes

- Added bundle policy semantics for `/vibe-init`: `automatic`, `custom`, and `off`.
- `/vibe-review` now distinguishes forgotten frontend utility gates from explicit opt-outs that lack rationale and replacement evidence.
- Extended `pendingRisk.status` to include `accepted`, `deferred`, and `closed-by-scope`.
- Updated preflight, sprint commit, dashboard, and project report behavior so only `open` pending risks are blocking/actionable by default.
- Strengthened Planner/Evaluator prompt policy for experience-led products: frontend, game, visual, canvas/WebGL/Three.js, editor, and dashboard Sprints need screenshot/playthrough/identity-payoff evidence.

## Migration

`1.7.3` normalizes legacy pendingRisk lifecycle aliases and adds a default `bundle.policy` to `.vibe/config.json` when missing.
