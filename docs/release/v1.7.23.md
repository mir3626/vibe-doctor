# v1.7.23 - PreCompact Handoff Auto-Refresh

Date: 2026-06-13

## Changes

- Added an opt-in `--auto-refresh` mode to `vibe-checkpoint.mjs` and wired it
  into the Claude Code `PreCompact` hook so context compaction never carries a
  stale or work-outdated handoff into the next window.
- The refresh triggers when the handoff is stale (not recently touched) or
  outdated (uncommitted changes exist, or new commits landed after
  `handoff.updatedAt`). When the handoff is already fresh and reflects current
  work, nothing is written.
- On trigger, the script upserts a bounded, idempotent
  `<!-- vibe:auto-state:start --> … <!-- vibe:auto-state:end -->` block into
  `.vibe/agent/handoff.md` with a mechanical git snapshot (branch, HEAD,
  changed files capped at 20, staged/unstaged diffstat, last 5 commits) and
  bumps `handoff.updatedAt`. The narrative remains Orchestrator-authored.

## Compatibility

- Without the `--auto-refresh` flag the script is byte-for-byte unchanged:
  `npm run vibe:checkpoint`, `/maintain-context`, and CI invocations stay
  read-only verification.
- The refresh is best-effort: it degrades silently when git is unavailable or
  state files are missing, and never blocks the normal checkpoint checks.
