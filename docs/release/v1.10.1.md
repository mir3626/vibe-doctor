# v1.10.1 - Shared-Module Ownership Boundary

Date: 2026-07-22

Adopts the osint-stock-screener proposal "shared-module ownership boundary":
the v1.9.0 downstream-migration guidance ("repoint the alias into the harness
tree, delete the local copy") inverted ownership of a project-owned
correctness module and is withdrawn. The vendoring itself stays — the harness
must run in any downstream and cannot import a project path.

## Changes

- **Ownership rule** (`docs/context/workflow-integrity.md` §11): the harness
  vendors and relatively imports its own copy (sync-replaced); a downstream
  that owns an equivalent module keeps it and its alias; only values both
  sides independently compute and compare must be byte-identical, and for
  those the downstream imports the sanctioned symbol only from the harness
  copy. Duplicated mechanics elsewhere are expected, pinned downstream by
  frozen conformance vectors.
- **Cross-boundary surface published** (compatibility contract; removal,
  rename, or semantic change is breaking — addition is not):
  - `.vibe/harness/src/universal-integrity-core/index.js` →
    `deriveFinalEvidenceManifest` (final-evidence manifest self-hash, FND-020)
  - `.vibe/harness/src/pro-roundtrip/report.js` → `workflowMatrixMarkdown`
    (matrix bytes feed `workflowMatrixSha256` in that manifest)

  The second symbol was undercounted in the proposal ("exactly one") — the new
  ownership audit found the live `workflowMatrixMarkdown` import in the
  downstream builder on its first run, and it is genuinely both-sides-compared,
  so it enters the surface rather than the warning list.
- **`vibe:sync-audit` ownership signal (report-only)**: warns
  (`harness-internal-import`) when product code imports `.vibe/harness/src/**`
  beyond the surface plus the project's `audit.harnessImportAllowlist`;
  sanctioned imports are listed as `cross-boundary-import` info. Never changes
  the exit code.
- **`vibe:sync-audit` drift signal (report-only)**: mirror pairs declared in
  `.vibe/config.json` `audit.sharedModuleMirrors` are hash-diffed and
  divergence is reported (`shared-module-drift`) without failing.
- **Guidance corrections**: `docs/release/v1.9.0.md` migration note corrected
  with a withdrawal marker; the `universal-integrity-core-resolution.test.ts`
  comment no longer tells downstreams to abandon their own module ("never to a
  second implementation" withdrawn); `docs/context/vibe-sync-audit.md`
  documents both signals.

## Compatibility

- No schema, protocol, or publisher behavior changes. The audit additions are
  warnings/info only; existing CI and preflight gates are unaffected.
- Downstreams with no equivalent module keep aliasing into the harness copy —
  that case is unchanged and remains the simplest correct choice.
- `audit.harnessImportAllowlist` and `audit.sharedModuleMirrors` are optional
  project-owned config keys; absent keys mean defaults (documented surface,
  no mirror diffing).
