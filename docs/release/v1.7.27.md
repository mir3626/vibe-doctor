# v1.7.27 - Hook Stdin Contract Auto-Detection

Date: 2026-07-14

## Changes

- Made all five Claude harness entrypoints recognize their hook event from the
  JSON payload on stdin. Explicit `--hook` and `--precompact-hook` flags remain
  supported, while legacy, cached, or partially synced commands that omit the
  flag now enter the same provider-safe hook mode.
- Added hook input `cwd` as a fallback when `CLAUDE_PROJECT_DIR` is unavailable.
  SessionStart, Notification, PostToolUse, Stop, and PreCompact therefore keep
  the project-root contract even when invoked from a stray host cwd.
- Kept manual CLI behavior unchanged for empty stdin or unrelated event JSON.
  PreCompact success emits no stdout; validation failures use stderr and exit 2.
  Stop QA failures still emit one valid JSON `systemMessage` and exit 0.
- Added regressions that invoke every hook without its explicit mode flag while
  sending the real Claude event shape through stdin.

## Root Cause

The v1.7.26 commands were correct when their explicit hook flags reached the
entrypoints. A legacy or partially applied command could still receive a valid
`PreCompact` payload on stdin without `--precompact-hook`; the script then used
manual mode and printed `[OK]` diagnostics to stdout. Claude Code correctly
rejected those lines as invalid PreCompact JSON. Event auto-detection removes
that remaining dependency on command-line flag propagation.

## Compatibility and Migration

- No data migration is required.
- Existing explicit hook commands remain canonical and unchanged.
- Downstream projects only need a normal harness sync. The input fallback also
  makes already-running legacy command forms safe once the scripts are updated.
