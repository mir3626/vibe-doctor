# v1.9.0 - Pro Roundtrip Byte-Exact Transport and Design-Owned Final Gate

Date: 2026-07-22

Adopts the osint-stock-screener downstream integrity hardening forced by the
GitHub Web Pro adversarial review rounds FND-019 (r7/r8), FND-020 (r8),
FND-022, FND-023, and FND-024 (downstream commits `702450b5..b3cd1578`), with
two upstream design decisions replacing the downstream reference shapes.

## Changes

- `runGitBinary` (worktree): bounded raw-`Buffer` git output. Blob bytes are
  never routed through a text encoding before validation, copy, or receipt.
- Pinned immutable transport (flow-store): `loadFlowSnapshot` pins ONE bridge
  commit and reads every flow/event byte from the Git object store —
  `ExactBlob { blobSha, byteSize, bytes }` readers, `ls-tree` event/file
  rosters, fatal UTF-8 + NUL rejection on the raw bytes. Mutable worktree
  files are never trusted; the bridge root must be a git checkout.
- Byte-exact packet copy (importer): payloads and the control documents
  `FLOW.json` / `COMPLETE.json` are copied as exact pinned blob bytes via
  `writeBytesAtomic` (no string round-trip, no re-serialization). Receipts
  bind payload and control-document blob identities and the copy fails closed
  on any receipt disagreement before state publication.
- Packet-state v2 (additive): every new sync writes
  `vibe-pro-packet-state-v2` with the `FLOW.json` / `COMPLETE.json` blob
  identities required; existing v1 packets remain readable so an open review
  round is never forced through a blocked re-sync.
- Bounded single reads (contract): `MAX_PACKET_FILE_BYTES` (16 MiB) and
  `readFlowFileOnce` (one descriptor-bound read) for local packet/state/
  checkpoint/manifest files; `EventPayloadAccessor` lets the flow transport
  inject the pinned-commit reader while plain-directory callers keep the
  previous behavior.
- Publisher evidence integrity (report): the publisher independently
  reconstructs the expected `FINAL-EVIDENCE-MANIFEST.json` from normalized
  checkpoint inputs plus the exact bytes being published, derives the
  product-to-current compare status from git facts, requires an empty
  skipped-check list, and requires canonical self-hash equality — a
  self-consistent rehash that drops a mandatory command, contract row, or
  checkpoint, or alters compare status or skipped checks, cannot publish.
- Design-owned final gate (FND-023, upstream shape): the frozen mandatory QA
  command roster is declared by the design event itself as a `CONTRACT.json`
  `finalGatePolicy: { mandatoryCommands: [...] }` block (optional in the
  schema). There is NO default roster and no harness-side registry: without
  the block the publisher refuses any final-evidence manifest as
  approval-eligible. `WEB-RUNBOOK.md` documents the block for Web Pro.
- Vendored `universal-integrity-core`: hash/shape/time profiles, bounded
  one-copy/single-read file primitives, exact-replay classification, and the
  `final-workflow-evidence-manifest-v1` codec ship inside the harness at
  `.vibe/harness/src/universal-integrity-core/` (dependency-free plain ESM +
  checked `.d.ts`, distributed by the existing `src/**` sync glob). The
  harness imports it relatively; downstream product code that must share the
  manifest derivation byte-for-byte aliases `#universal-integrity-core` to
  this path.
- Test fixtures moved to harness-owned
  `.vibe/harness/test/fixtures/pro-roundtrip/` (tests previously read
  project-owned `docs/plans/**` paths that do not survive a downstream sync).
  New `universal-integrity-core` verification group (7 → 8); contract tests
  cover pinned-object tamper resistance, non-ASCII byte equivalence,
  invalid-UTF-8/NUL fail-closed loads, control-document byte exactness,
  publisher reconstruction, rehash-forgery rejection, and the
  `finalGatePolicy` schema and enforcement.

## Compatibility

- Pro protocol event/flow schemas are unchanged except the additive optional
  `finalGatePolicy` block in `CONTRACT.json`; existing contracts without it
  keep parsing, and flows that never generate `FINAL-EVIDENCE-MANIFEST.json`
  are unaffected.
- `loadFlowSnapshot` now requires the bridge root to be a git checkout (the
  prepared bridge worktree always is). Plain-directory fixtures must become
  committed trees.
- Packet state: v1 stays historical-readable; every new write is v2.
- The vendored core is loaded from `src` by the tsx runtime; the optional
  `vibe:build` dist output does not include the plain-ESM core files.
- Downstream migration (osint-stock-screener): after syncing v1.9.0, delete
  the orphaned `src/pro-roundtrip/final-gate-policy.ts` if the sync leaves it
  behind, and have Web Pro publish a design revision whose `CONTRACT.json`
  declares the frozen roster as `finalGatePolicy` before the next manifest
  publication.
  > **Correction (v1.10.1)** — this note originally also instructed the
  > downstream to repoint its `#universal-integrity-core` alias into the
  > harness tree and delete its local `shared/universal-integrity-core/` copy.
  > That guidance inverted ownership of a project-owned module and is
  > withdrawn. The correct pattern is the shared-module ownership boundary in
  > `docs/context/workflow-integrity.md` §11: a downstream that owns an

  > `workflowMatrixMarkdown`) from the harness copy. A downstream with no such
  > module aliases into the harness copy as before.
