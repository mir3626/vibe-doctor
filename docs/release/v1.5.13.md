# v1.5.13

## Summary

Prevents dashboard/report browser-launch failures from terminating WSL or Linux sessions.

## Changes

- `npm run vibe:dashboard` now handles async browser opener errors such as `spawn xdg-open EACCES` as warnings.
- `npm run vibe:report` uses the same error handling for `xdg-open`/`open`/`cmd start` failures.
- Added regression coverage for child process `error` events emitted after browser opener spawn.

## Verification

- `npm run typecheck`
- `node --import tsx --test test/dashboard-server.test.ts test/project-report.test.ts`
