# v1.5.1

## Summary

Provider-neutral lifecycle support for downstream projects that use Claude, Codex, Gemini, or another CLI agent.

## Changes

- Added `scripts/vibe-agent-session-start.mjs` as the canonical session-start entrypoint.
- Wired Claude `SessionStart`, `scripts/run-codex.sh`, and `npm run vibe:run-agent` through the shared session-start entrypoint.
- Added `npm run vibe:session-start` and `npm run vibe:checkpoint`.
- Added `_common-rules.md` Section 16 for provider-neutral context persistence.
- Added default `.vibe/config.json.upstream` so downstream projects can detect harness updates with `vibe-version-check`.

## Notes

Codex still does not expose a true `PreCompact` hook equivalent to Claude Code. For Codex and other CLI providers, the harness enforces the portable fallback: update `.vibe/agent/handoff.md` and `.vibe/agent/session-log.md`, then run `node scripts/vibe-checkpoint.mjs`.
