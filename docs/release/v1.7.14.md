# v1.7.14 - Template Project-State Hygiene

Date: 2026-05-11

LTS: `v1.7.3-lts` remains the stable baseline. This is a mainline init/sync hygiene patch.

## Changes

- Reset project-owned context shards to explicit `PROJECT NOT INITIALIZED` placeholders:
  - `docs/context/product.md`
  - `docs/context/architecture.md`
  - `docs/context/conventions.md`
  - `docs/context/qa.md`
  - `docs/context/secrets.md`
  - `docs/context/tokens.md`
- Removed checked-in upstream project artifacts from the template:
  - stale `docs/plans/*.md` dogfood/upstream plans
  - stale `docs/prompts/*.md` Sprint prompts
  - generated `docs/reports/project-report.html`
  - archived `.vibe/archive/prompts/*.md` Sprint prompts
- Reset checked-in project runtime snapshots to template placeholders:
  - `.vibe/agent/sprint-status.json`
  - `.vibe/agent/handoff.md`
  - `.vibe/agent/session-log.md`
- Extended `/vibe-init` copied-template cleanup to reset:
  - `.vibe/agent/project-map.json`
  - `.vibe/agent/sprint-api-contracts.json`
  - `.vibe/agent/tokens.json`
  - `.vibe/agent/project-decisions.jsonl`
  - project prompt/report/archive directories
  - ignored runtime directories such as `.vibe/agent/daily` and `.vibe/runs`
- Added `.vibe/harness/migrations/1.7.14.mjs` to remove known copied template artifacts from existing downstream projects without deleting project-owned files that merely reuse common names.
- Added template hygiene regression tests.
- Updated sync/init guards so explicit `PROJECT NOT INITIALIZED` placeholders still block `/vibe-sync` until real product context exists.

## Why

Some project-owned files still contained upstream `vibe-doctor` plans, prompts, reports, or conventions. New projects should not inherit those assumptions during `/vibe-init`, especially because conventions are intentionally preserved outside marker blocks after initialization.

## Compatibility

- Project-owned downstream files are preserved unless they match known upstream template artifact markers.
- The new placeholders intentionally keep `/vibe-sync` blocked until `/vibe-init` writes real project context.
