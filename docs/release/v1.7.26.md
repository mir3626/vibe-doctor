# v1.7.26 - Claude Hook CWD and Output Contract Hardening

Date: 2026-07-14

## Changes

- Bound all five Claude Code harness hooks to `${CLAUDE_PROJECT_DIR}` in
  `.claude/settings.json`, so SessionStart, Notification, PostToolUse, Stop,
  and PreCompact no longer depend on the host process cwd. The PostToolUse
  command also uses `npm --prefix` and the verified `run --silent <script> --
  --hook` argument order.
- Added explicit hook modes. Successful and disabled hook paths emit no
  stdout; reportable nonblocking results emit one valid JSON object with a
  `systemMessage`. PreCompact validation failures emit stderr only and exit 2.
- Kept Stop deliberately nonblocking: a failed `vibe:qa` run is stored under
  `.vibe/runs/<date>/stop-qa-*.log`, reported as JSON, and returns exit 0 from
  the hook. Manual invocation retains its diagnostic nonzero QA exit status.
- Added stray-cwd, empty-stdout, valid-JSON, npm forwarding, kill-switch, and
  settings-wiring regressions for every harness hook entrypoint.

## Compatibility and Migration

- No data migration is required. Hook scripts and the `hooks` section of
  `.claude/settings.json` are already harness-owned sync surfaces.
- Direct CLI behavior is unchanged unless `--hook` or `--precompact-hook` is
  supplied. Config audit now reports a failed `git ls-files` call instead of
  silently treating it as a clean repository.
- `VIBE_HARNESS_HOOKS=off` remains an immediate exit-0 kill-switch and is
  silent in hook mode.
