# v1.6.9

## Summary

Wires the dashboard attention pipeline for Claude and Codex so permission prompts, completed Claude turns, and completed Codex wrapper runs can surface as browser/dashboard notifications.

## Changes

- Added Claude Code `Notification` hook wiring for `permission_prompt`, `idle_prompt`, and `elicitation_dialog`.
- Refactored `vibe-attention-notify.mjs` to reuse the shared `appendAttentionEvent()` writer.
- Extended `vibe-attention.mjs` with provider metadata, non-urgent `info` severity, and optional payload support.
- Added Codex wrapper attention events on successful and failed `run-codex` executions for both Bash and CMD wrappers.
- Kept Codex permission-prompt support wrapper-based: Codex CLI does not expose a Claude-style project hook, so the harness records wrapper completion/failure events where it can.
- Cleared the v1.6.8 wiring drift findings for `vibe-attention.mjs` and `vibe-attention-notify.mjs`.
