# v1.8.0 — Web Pro Bridge

웹 ChatGPT Pro 세션 ↔ CLI 왕복 브릿지. 마지막 goal 구현을 웹 Pro 리뷰로 보내고(`$vibe-goal-audit`), 신규 기능 설계를 요청하고(`$vibe-pro-design`), 결과 패키지를 `docs/plans/<folder>/`에 검증·원자적으로 설치한다. 정본 설계: `docs/plans/web-pro-bridge/design.md` (Hybrid v2 — Claude 설계와 GPT Pro 세션 설계 패키지 `vibe-pro-bridge-design/`의 병합, §2 채택 매트릭스 참조).

## 신규 표면

- **스킬**: `$vibe-goal-audit`, `$vibe-pro-design` (.claude 공유 runbook + .codex wrapper)
- **커맨드**: `vibe:pro-audit` / `vibe:pro-design` / `vibe:pro-sync` / `vibe:pro-status` / `vibe:pro-mcp` / `vibe:pro-apply` (`vibe-pro-bridge.mjs` 단일 엔트리)
- **모듈**: `.vibe/harness/src/pro-bridge/` — goal-source provider 체인(Codex App Server stub → vibe-goal-iterate → handoff → git 재구성, confidence 라벨), GitHub visibility gate + secret-safe patch, A~I 리뷰 프롬프트 composer(리뷰 12차원, 커넥터 제약 경고, 프롬프트 인젝션 방어), vibe-bundle v1 파서, 원자적 result importer(allowlist·SHA-256 바인딩·충돌 규칙·provenance), local-first MCP mailbox(12-tool streamable-HTTP, chunked upload, lifecycle/idempotency/revision chain, single-tenant 토큰, cloudflared/ngrok 터널 helper), transports(manual / mcp-mailbox / workspace-agent / responses-api)
- **web-origin**: 웹 Pro에서 먼저 설계 시작 → `vibe:pro-sync --latest`로 CLI 반입
- **schemas**: `pro-bridge-{goal-source,review-request,review-result}.schema.json` (gen-schemas 관리)
- **문서**: `docs/context/pro-bridge-setup.md` (ChatGPT Developer Mode 1회 등록·터널·트러블슈팅), CLAUDE.md 스킬/훅 표

## 보안·경계

- 브라우저 DOM 자동화·자동 제출·모델 피커 자동화 없음 (ToS 준수 — 딥링크 프리필과 Developer Mode MCP만 사용)
- 자동 git push 없음, GitHub write 없음, repo 소스 미러링 없음, 결과 자동 구현 없음
- 쓰기 표면은 `docs/plans/<folder>/`(allowlist)와 `.vibe/pro-bridge/`(gitignored 스테이징)뿐 — 하네스 state와 lifecycle hook 무결합, 미사용 시 오버헤드 0
- `proBridge.enabled` opt-in (기본 false). 전송 전 외부 발행 1회 고지 + `--yes` 게이트

## 부수 수정

- `vibe-agent-session-start.mjs`가 비-hook 실행에서 stdin을 drain해 run-codex.sh 파이프 프롬프트가 소실되던 회귀(v1.7.26~27) 수정 + wrapper `</dev/null` 이중 방어
- MCP 기본 포트 8848 → **18488** (Windows WinNAT excluded port range 8827–8926 실측 충돌 회피). `EACCES`/`EADDRINUSE` 시 netsh 확인법 + 오버라이드 안내 출력
