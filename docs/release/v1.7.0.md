# v1.7.0 - Harness Source Boundary

## Changes

- Moved harness runtime, tests, migrations, TypeScript configs, and Playwright config under `.vibe/harness/**`.
- Kept root `src/**`, `scripts/**`, `test/**`, `app/**`, `components/**`, and `lib/**` project-owned for downstream projects.
- Updated package scripts so synced harness entrypoints are `scripts.vibe:*`; product scripts such as `test`, `build`, `typecheck`, and `test:ui` are no longer harness-owned by sync.
- Added migration `1.7.0` to remove legacy root harness files only when sync hashes prove they are unmodified, update the Codex provider wrapper path, and redirect old harness-owned `test:ui` aliases to `npm run vibe:test-ui`.
- Updated bootstrap to expand glob manifest entries and kept `scripts/vibe-sync-bootstrap.mjs` as a compatibility bridge for legacy raw-URL bootstrap instructions.
- Clarified `/vibe-review` as a template/harness review rather than ordinary downstream product code review.

## Verification

- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run vibe:test-ui -- --version`
- `npm run vibe:test-ui`
- `npm run vibe:config-audit --silent`
