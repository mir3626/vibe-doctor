# v1.6.0

## Summary

Adds universal creative-completion directives after the dogfood11 autochess run exposed a pass-only bias: the generated product satisfied mechanical checks but failed the user's genre and identity expectations.

## Changes

- Interview synthesis now probes reference products or cases and asks how the product's visual, ergonomic, or conceptual identity should differ.
- Sprint planning now requires a user-visible closure sentence and can reject horizontal technical slices in favor of vertical usable slices.
- Orchestration guidance now separates mechanical checks from inspection/demo acceptance criteria instead of dropping hard-to-automate product-quality criteria.
- Generator common rules now treat AC as a lower bound and require `## Creative bets` reporting for scope-contained agent-initiated differentiation.
- Soft-freeze LOC caps are explicitly harness-only; product/downstream Sprint scope should be shaped by user value and quality, not arbitrary LOC, file-count, or bundle-size caps.
- Added `gap-pass-only-product-identity` to the harness-gaps ledger as a partial fix pending dogfood11 iter-A2 validation.

## Migration Notes

- No state migration is required.
- Existing downstream `bundle` and `browserSmoke` settings remain project-owned and are not overwritten by sync.
- Teams should expect future Sprint prompts and Generator reports to include `AC (lower bound)` and `Creative bets` sections.
