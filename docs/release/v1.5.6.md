# v1.5.6

## Summary

Executable-mode correction for Linux shell wrappers during sync.

## Changes

- Kept synced `.sh` harness files executable with mode `0755`.
- Kept regular synced files non-executable by preserving the existing mode or defaulting new files to `0644`.
- Added regression coverage so Windows-mounted templates cannot make regular files executable while `scripts/run-codex.sh` remains runnable on Linux/WSL.

## Notes

This is a follow-up to v1.5.5. The mode-safe copy fix must distinguish executable shell wrappers from regular Markdown, JSON, TypeScript, and config files.
