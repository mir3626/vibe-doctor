# v1.6.4

## Summary

Clarifies Codex Orchestrator context persistence as an explicit maintain-context workflow instead of pretending Codex has a native PreCompact or context-threshold hook.

## Changes

- Expanded `maintain-context` into a concrete workflow for updating `handoff.md`, appending `session-log.md`, and running `npm run vibe:checkpoint`.
- Documented that Sprint Generator Codex runs normally do not need token-threshold checkpoint automation because they are short-lived and return state through completion reports.
- Documented that Codex used as the main Orchestrator must checkpoint at work boundaries such as decisions, reviews, releases, pushes, syncs, and final handoff.
- Updated provider-neutral common rules and Codex execution docs to distinguish Codex Orchestrator operation from `run-codex.sh` Generator invocations.
- Changed `vibe-checkpoint` failure wording from PreCompact-specific to provider-neutral checkpoint wording.
