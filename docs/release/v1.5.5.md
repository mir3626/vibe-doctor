# v1.5.5

## Summary

Project-safe sync policies for common customizable harness files.

## Changes

- Added `replace-if-unmodified` for files that should update only when the downstream copy has not been customized.
- Added `json-array-union` for JSON arrays such as VS Code extension recommendations.
- Added marker-based section merge ownership for `AGENTS.md` and `GEMINI.md`.
- Moved `.env.example`, `.github/workflows/ci.yml`, `.vscode/*.json`, `.editorconfig`, and `.gitattributes` out of raw replacement where needed.
- Normalized synced file modes so copying from Windows-mounted templates does not make regular files executable on Linux/WSL.
- Added regression coverage for preserve-if-modified behavior, JSON array union, marker bootstrapping, and mode-safe copy.

## Notes

Downstream projects can keep project-specific environment templates, CI workflows, editor settings, and agent memory custom sections without blocking future harness syncs.

When validating a Windows-mounted checkout from WSL, use a Linux-installed `node_modules` tree. Native packages such as `esbuild` cannot be shared between Windows and Linux.
