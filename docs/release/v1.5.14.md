# v1.5.14

## Summary

Lets legacy or partially bootstrapped projects run `/vibe-sync` even when `.vibe/config.json` has no `upstream` entry.

## Changes

- `/vibe-sync` now initializes missing upstream config before cloning:
  - existing upstream config is preserved;
  - vibe-doctor template/fork origins are used for dogfood clones;
  - product repository origins fall back to `https://github.com/mir3626/vibe-doctor.git`;
  - template source checkouts are marked `self` and still skip self-sync.
- `vibe-version-check` uses the same upstream inference so SessionStart does not store product repo origins as harness upstreams.
- `vibe-sync-bootstrap` can now create `.vibe/config.json` for legacy projects that lack the shared config file.
- Added regression coverage for missing-upstream sync, product-origin fallback, self-checkout guard, and config-less bootstrap.
