# v1.6.11 - Linux CI run-codex wrapper tests

## Summary

- Fixed the GitHub Actions `npm test` failure introduced in `v1.6.1`.
- Resolved the POSIX test harness issue where `PATH` was intentionally narrowed and `bash` was no longer discoverable on Linux runners.
- Added `CODEX_BIN` support to `scripts/run-codex.sh` so tests and advanced users can point the wrapper at an explicit Codex binary path.
- Updated the WSL Windows npm-shim regression test to exercise the actual rejected path shape on Linux CI instead of relying on the host checkout path.

## Verification

- `node --import tsx --test test/run-codex-wrapper.test.ts test/shell.test.ts` on Windows
- `node --import tsx --test test/run-codex-wrapper.test.ts test/shell.test.ts` on WSL after `npm ci`
