# v1.13.1 - `go` Skips Flows on a Superseded Protocol Generation

Date: 2026-07-23

Adopts the downstream handoff (P2): after v1.12.0 content-addressed protocol
namespaces, a NON-CLOSED flow pinned to a retired generation (e.g. `v1`)
permanently poisoned bare `vibe:pro-go go` — `selectGoFlow` filtered only on
branch/repository/closed, picked the drifted flow, and `syncFlow` →
`verifyPinnedProtocol` hard-errored on every invocation, making every other
flow (and "nothing to do") unreachable. Such flows can also never be CLI-closed
(close requires a latest Pro approval; every CLI op protocol-gates), so they
are permanent honest ledger entries the selector must tolerate.

## Changes

- `selectGoFlow` loads the local protocol once per invocation and skips any
  auto-selection candidate whose pinned `flow.protocol.version` differs from
  the locally computed version — exactly the condition `verifyPinnedProtocol`
  throws on. Skips are recorded, never silent.
- `goCommand` emits `skippedIncompatibleFlows`
  (`[{ flowPath, pinnedVersion }]`, `[]` default) in its JSON result, so a
  genuinely active flow that drifted across a sync stays visible.
- When EVERY non-closed candidate is protocol-incompatible, bare `go` throws an
  enriched selector error naming each skipped flow and its pinned version plus
  the local generation — instead of the raw protocol mismatch of whichever
  flow happened to rank newest. With zero candidates the original
  `no non-closed Pro flow matches …` message is unchanged.
- Explicit targets are NEVER skipped: `go <flowPath>`, `status`, `sync`,
  `report`, `accept-review`, `close` still route through
  `verifyPinnedProtocol` and fail closed with the precise generation error.

## Known residue (out of scope, recorded)

Bare `status` resolves its flow through `resolveFlowPath` (lexicographic
latest, no branch/repo/closed filter, no protocol filter) and can still land on
a foreign-generation flow; it stays fail-closed with the precise error. Fix on
next touch of that resolver if dogfood friction shows.

## Verification

- `vibe:typecheck` clean; focused pro-roundtrip CLI suite green including:
  compatible+foreign flow pair → `go` selects the compatible one and lists the
  other under `skippedIncompatibleFlows`; only-foreign flow → enriched
  "no operable non-closed Pro flow … superseded protocol generation" error
  (not the raw mismatch); explicit `go <foreignFlow>` → unchanged raw
  generation error.
- Full `vibe:self-test`: 556 tests — 555 pass / 0 fail / 1 intentional skip.
  (A first run showed 2 statusline flakes under parallel load — transcript-read
  fail-soft omitting the Claude segment; isolated suite 18/18 and the full
  rerun green, same acceptance pattern as the v1.9.0 EBUSY precedent.)
- ~+25 src LOC in one existing command module — inside the soft-freeze cap, no
  waiver needed.
