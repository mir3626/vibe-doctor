# v1.7.13 - Template Iteration State Reset

Date: 2026-05-08

LTS: `v1.7.3-lts` remains the stable baseline. This is a mainline init/sync migration patch.

## Changes

- Reset upstream template-owned `.vibe/agent/iteration-history.json` to an empty history.
- Reset upstream template-owned `docs/plans/sprint-roadmap.md` to an initial placeholder instead of carrying historical upstream Iteration 7-12 roadmap content.
- Updated `/vibe-init` so copied template sprint state cleanup also resets iteration history and sprint roadmap.
- Added `.vibe/harness/migrations/1.7.13.mjs` to remove known copied upstream iteration entries such as `iter-9` from downstream projects.
- Added regression coverage for init cleanup and migration behavior, including a guard that preserves a real project-owned `iter-9`.

## Why

Some downstream projects inherited upstream template iteration history and roadmap content. When `/vibe-iterate` looked at that copied state, the next iteration could start at stale upstream numbering such as `iter-9` instead of the downstream project's first iteration.

## Compatibility

- Project-owned iteration history is preserved unless entries match known upstream template labels and sprint IDs.
- Project-owned roadmaps are preserved when their iteration sections are not the known upstream Iteration 7-12 template set.
- Existing downstream projects should sync to `v1.7.13` to run the guarded cleanup migration.
