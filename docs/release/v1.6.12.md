# v1.6.12 - downstream init and review signal hardening

## Summary

- Fixed provider command lookup so `commandExists()` honors provider-specific `cwd` and `env.PATH` before `vibe:run-agent` rejects a custom CLI.
- Added a Codex initialization boundary to agent memory and common rules. Downstream clones with missing or template-owned `product.md` / `sprint-status.json` must run `/vibe-init` before product or maintenance work.
- Changed `/vibe-review` frontend opt-in detection to use explicit platform fields or `PROJECT/HARNESS:review-signals` marker blocks instead of scanning arbitrary product prose.
- Replaced the synced `test:ui` package script with `scripts/vibe-playwright-test.mjs`, which prints local install guidance when `@playwright/test` is absent and delegates to the local Playwright CLI when present.

## Verification

- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run test:ui -- --version`
