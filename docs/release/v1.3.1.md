# v1.3.1 (2026-04-16) — Dead code cleanup + statusline 버전 표기

## Highlights

### Sprint M13 — dead code + wiring gap 전면 정리
- `scripts/vibe-sprint-complete.mjs` 의 `archiveSprintPrompts` 필터 버그 수정 — 이전 `entry.startsWith(${sprintId}-)` regex 가 `sprint-M5-native-interview.md` 같은 실제 파일명을 매치 못해 M1~M12 아카이빙 전량 실패. 이제 `base === sprintId || base.startsWith(${sprintId}-)` 로 exact + suffix 둘 다 커버.
- `scripts/run-claude.{sh,cmd}` 삭제 — 실 호출처 0 인 "not wired" stub. 미래 provider 편입 시 재생성.
- `docs/prompts/` 의 24개 orphan sprint prompt 를 `.vibe/archive/prompts/` 로 retroactive `git mv`.
- `CLAUDE.md`: `/vibe-iterate` + `scripts/vibe-project-report.mjs` 를 Sprint flow + 관련 스킬 목록에 명시 (v1.3.0 에서 누락됐던 wiring gap).
- `docs/context/harness-gaps.md`: archive-regex / v1.3-wiring / dead-stub / review-catch-wiring-drift 4건 gap 기록.
- `test/vibe-sprint-complete.test.ts` 신규 — archive regex 회귀 방지 (exact / suffix / miss 3케이스).

### Sprint M14-A — Statusline 하네스 버전 + 업데이트 힌트
- `.claude/statusline.sh` + `.claude/statusline.ps1` 출력 끝에 버전 정보 suffix 추가.
- 조건별 suffix:
  - 버전 정보 없음: suffix 없음 (기존 동작)
  - 설치 버전만 있음: ` | v1.3.1`
  - 설치 버전 < 최신 캐시: ` | v1.3.1 ⚠ v1.4.0 (/vibe-sync)`
- 데이터 소스: `.vibe/config.json.harnessVersionInstalled` (fallback `harnessVersion`) + `.vibe/sync-cache.json.latestVersion` (vibe-version-check.mjs 24h 갱신).
- `test/statusline.test.ts` +4 regression 케이스.

### 기타 바로잡기
- `.claude/settings.json` 의 `statusLine` 엔트리 등록 — v1.3.0 에서 스크립트는 만들었지만 Claude Code 가 호출하도록 wiring 안 되어 있던 gap 해소 (M9 silent gap).
