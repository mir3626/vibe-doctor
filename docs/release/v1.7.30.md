# v1.7.30 - Session Lifecycle and Hidden QA Hardening

Date: 2026-07-14

## Changes

- Added an atomic short-window SessionStart delivery claim under ignored
  `.vibe/runs/session-start-deliveries/`. Repeated hook deliveries with the same
  `session_id + source` within 60 seconds run once, while `startup`, `resume`,
  `clear`, and `compact` source transitions remain independent.
- Added `sessionId`, `source`, and `invocation` provenance to
  `session-started` daily events so future duplicate reports can be correlated
  to a concrete lifecycle delivery path.
- Isolated Stop harness QA from lifecycle hooks by removing
  `CLAUDE_PROJECT_DIR` and forcing `VIBE_SKIP_AGENT_SESSION_START=1` for both
  `vibe:typecheck` and `vibe:self-test`. Dedicated SessionStart tests explicitly
  opt back in.
- Replaced the nested `spawnSync('npm', ..., { shell: true })` path with direct
  Node execution of npm's CLI when available. The portability fallback uses a
  hidden Windows command processor, and every nested QA child sets
  `shell: false` plus `windowsHide: true`.
- Applied `windowsHide: true` to provider-neutral SessionStart children, daily
  event recording, optional dashboard startup, and the TypeScript run-agent
  lifecycle bridge.

## Root Cause

v1.7.29 correctly detached and hid the outer Stop worker, but the worker then
started npm through a new shell without the Windows hidden-window option. That
nested shell could become a visible foreground terminal. Its self-test process
also retained normal SessionStart activation, so lifecycle-aware test paths
could write repeated `session-started` entries to the real project instead of
remaining isolated fixtures.

The static Claude settings contained only one SessionStart registration; the
observed burst came from nested QA execution and missing lifecycle isolation,
not duplicate hook configuration.

## Compatibility and Migration

- No database or project-state migration is required.
- SessionStart delivery markers and Stop QA receipts remain runtime-only under
  ignored `.vibe/runs/`.
- Provider-wrapper invocations without a Claude hook `session_id` are not
  deduplicated. The explicit `VIBE_SKIP_AGENT_SESSION_START=1` contract remains
  the control for nested provider or test execution.
- Downstream behavior outside hook execution is unchanged.

## Verification

- Focused SessionStart tests for lifecycle provenance, repeated-delivery
  suppression, source-transition preservation, kill-switch behavior, and
  explicit nested-provider skip
- Focused Stop gate tests for hidden shell-free child policy, lifecycle env
  isolation, detached work deduplication, success caching, and failure receipts
- Upstream bootstrap regression with explicit lifecycle-test opt-in

- `npm run vibe:build`
- `npm run vibe:self-test`
- `npm run vibe:sync-audit`
- `npm run vibe:checkpoint`
