# v1.6.7

## Summary

Adds rule disposition reporting to `vibe-rule-audit` so imperative rules can be triaged before being promoted to hard script gates.

## Changes

- `vibe-rule-audit` now reports each rule's disposition as `covered`, `pending`, `manual-review`, `delete-candidate`, or `undisposed`.
- Added `summary.disposed`, `summary.undisposed`, and `summary.byDisposition` to JSON output.
- Text output now includes an explicit "Undisposed" section and disposition counts.
- Added `--fail-on-undisposed` for projects that want to turn undisposed rules into a hard gate.
- Updated `gap-rule-only-in-md` from pending-only scanner status to partial disposition-gate coverage.

## Verification

- `node --import tsx --test test/rule-audit.test.ts`
- `node scripts/vibe-rule-audit.mjs --format=json`
- `node scripts/vibe-rule-audit.mjs --fail-on-undisposed` returns exit 1 while current undisposed rules remain
