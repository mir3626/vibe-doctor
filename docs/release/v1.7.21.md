# v1.7.21 - Init Readiness Gate and Quieter Downstream QA

Date: 2026-06-02

## Changes

- Added `npm run vibe:init-ready`, a pre-MVP readiness gate for delegated
  agent-mode `/vibe-init` sessions. It blocks Sprint prompts and implementation
  until Phase 2-4 artifacts are project-owned.
- Agent delegation prompts now require direct phase-shard reading and an
  explicit `vibe:init-ready` pass before Generator handoff or MVP work.
- Downstream `vibe:qa` now skips root QA scripts that still point at
  harness-owned aliases such as `vibe:self-test`, `vibe:typecheck`,
  `vibe:build`, and `vibe:test-ui`.
- `vibe-stop-qa-gate` now captures full `vibe:qa` stdout/stderr under
  `.vibe/runs/<date>/stop-qa-*.log` and prints only concise ok/fail summaries.

## Compatibility

- Existing downstream projects can keep project verification in root
  `test`/`typecheck`/`lint`/`build` scripts. Harness verification remains
  available through explicit `vibe:*` commands.
- Agent-mode one-liner initialization now fails fast before MVP work if
  `.vibe/config.local.json`, project context, interview logs, roadmap, or git
  initialization are missing or still template-owned.
