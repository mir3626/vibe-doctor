# v1.5.4

## Summary

Project-safe `.gitignore` syncing.

## Changes

- Added a `line-union` sync strategy for line-oriented files.
- Moved `.gitignore` from raw harness replacement to hybrid line merge.
- Preserved downstream project ignore entries while still appending new upstream harness ignore entries.
- Added regression coverage for `.gitignore` line merge planning and merge output.

## Notes

This prevents project-specific entries such as `runtime/` from causing `.gitignore` conflicts or being overwritten during future `vibe:sync` runs.
