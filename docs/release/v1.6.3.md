# v1.6.3

## Summary

Clarifies `upstream.ref` semantics so exact refs are hard pins and caret refs opt in to compatible automatic harness updates.

## Changes

- Exact `upstream.ref` values such as `v1.5.15` are preserved as hard pins during `/vibe-sync` and `npm run vibe:sync`.
- Floating refs use package-style caret syntax: `^v1.5.15` or `^1.5.15` resolves to the latest compatible cached harness tag.
- Unpinned projects still track the cached latest harness tag.
- Explicit sync now runs `vibe-version-check.mjs --force`, so a fresh `.vibe/sync-cache.json` cannot hide a newly published harness tag.
- The version cache now records all discovered harness semver tags, allowing caret ranges to avoid crossing major or zero-major semver boundaries.
- Statusline and session-start update notices do not nag projects that are intentionally pinned to an exact ref.
- `/vibe-sync` skill documentation now describes exact, caret, branch, and unpinned ref resolution.
