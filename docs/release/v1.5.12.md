# v1.5.12

## Summary

Restores `upstream.ref` as an explicit pinned harness version while keeping a safe update path for stale pins.

## Changes

- `/vibe-sync` now preserves a semver `upstream.ref` by default.
- When no `--ref` is supplied and the version cache shows a newer harness tag, interactive sync asks whether to keep the pin, update once to the latest tag, or cancel.
- Non-interactive sync keeps the pin and prints the `--ref` command needed to bypass it.
- Unpinned projects still follow the cached latest harness tag when available.
- `scripts/vibe-sync-bootstrap.mjs` now preserves existing pins but stops creating a new `upstream.ref` automatically for unpinned projects.
- Added regression coverage for pinned ref resolution and legacy bootstrap behavior.
