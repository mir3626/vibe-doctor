# v1.7.1 - Codex Init Agent-Mode Guard

## Changes

- Added an explicit `/vibe-init` `--mode=human|agent` gate for agent-skill execution.
- Made `--mode=agent` delegation-only: it records `.vibe/config.json.mode = "agent"`, renders a runtime-specific delegation prompt, and exits before Phase 1-1 bootstrap state is created.
- Updated Codex delegation prompts to prioritize `AGENTS.md`, `docs/context/orchestration.md`, and `docs/context/codex-execution.md` while treating `CLAUDE.md` as shared nominal charter context.
- Fixed `.vibe/config.local.example.json` so generated Codex provider config uses `./.vibe/harness/scripts/run-codex.sh`.
- Clarified `/vibe-review` dependency handling and the narrow read-only exception for init/bootstrap/harness process failure reviews in partial checkouts.
