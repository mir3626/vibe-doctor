# v1.6.2

## Summary

Fixes Codex agent role TOML escaping and makes agent role files part of harness sync.

## Changes

- Canonical mojibake detection commands now use `[?]` for a literal question mark instead of `\?`, so the command is safe inside TOML basic strings as well as Markdown and shell prompts.
- `.codex/agents/**` is now a harness-owned sync path, allowing downstream projects to receive fixed Codex agent role definitions.
- Added a migration that converts legacy `developer_instructions = """ ... """` agent prompt bodies to TOML multiline literal strings when safe, while preserving existing prompt text.
- Added regression tests that validate upstream Codex agent TOML files and the legacy migration path without adding a TOML parser dependency.

## Migration Notes

- Existing downstream projects with malformed `.codex/agents/coder.toml` should run `npm run vibe:sync -- --ref v1.6.2`.
- If sync reports `.codex/agents/*.toml` as a conflict because the file predated sync hashes, accept the upstream version or run with `--force`; the migration also repairs the known malformed prompt body in place.
