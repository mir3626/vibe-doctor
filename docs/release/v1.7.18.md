# v1.7.18 - Active Roadmap Lifecycle

Date: 2026-05-17

## Changes

- Added `vibe-roadmap-maintenance.mjs` to keep `docs/plans/sprint-roadmap.md`
  focused on the current iteration while archiving inactive iteration sections
  to `docs/plans/archive/roadmaps/`.
- Wired roadmap lifecycle checks into sprint completion, preflight warnings, and
  a guarded sync migration.
- Expanded roadmap sprint-id parsing to support heading-style ids such as
  `### iter-28-sprint-01-...` in preflight, sprint completion, dashboard, and
  project report paths.
- Project reports now read archived roadmap slices so historical sprint details
  remain visible after active roadmap compaction.
- Added a checkpoint budget for `.vibe/agent/handoff.md`; oversized active
  handoff files now fail checkpoint with remediation guidance instead of
  silently becoming long-term context bloat.

## Compatibility

- Append-only `session-log.md` remains append-only. It is treated as durable
  history, not an active context file.
- Existing roadmap content is archived before the active file is rewritten.
- If an archived iteration file already exists with different content, the
  maintenance command writes a timestamped revision instead of overwriting it.
