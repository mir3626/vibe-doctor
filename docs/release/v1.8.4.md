# v1.8.4 - Contract-less Remediation Reports and Piped-Prompt Guard

Date: 2026-07-20

## Changes

- Fixed `recordSprintReport` so a design-less (contract-less) audit flow expects
  `reportKind: "remediation"` once the latest bridge event is feedback, matching
  the documented CLI contract and the publisher, which only reads
  `remediation/<feedback-event>/` checkpoints in that state. Previously the
  recorder demanded `audit` evidence that the publisher could never see, making
  the flow's `NNNN--codex--remediation-report--rNN` target unreachable.
- Removed the redundant `reportKind !== 'audit'` clause from the contract-less
  binding guard. Kind matching stays fail-closed via the expected-kind check;
  null design/Sprint binding enforcement is unchanged.
- Fixed `run-codex.sh` so the provider-neutral session-start lifecycle child
  runs with stdin redirected to `/dev/null`. Since v1.7.27 the session-start
  hook auto-detects its Claude event by reading fd 0, which silently drained
  piped prompts (`cat prompt.md | run-codex.sh -`) and made every piped
  Generator delegation fail with `No prompt provided via stdin.`.
- Added a contract-less audit flow regression test covering remediation
  checkpoint placement under `remediation/<feedback-event>/AUDIT/`, rejection of
  `audit`/`implementation` kinds after feedback, rejection of non-null
  design/Sprint bindings, the unresolved-P1 publication block, and successful
  `remediation-report` publication.
- Added a wrapper regression test that pipes a prompt past a session-start stub
  that reads fd 0 and asserts codex still receives the full prompt.

## Compatibility

- No Pro protocol, schema, flow archive, default-branch, or confirmation policy
  changed in this release.
- Contract-bearing (design) flows behave identically; only the previously
  unreachable contract-less remediation path changed.
- Downstream repos that applied the identical local patch to
  `.vibe/harness/src/pro-roundtrip/report.ts` (e.g. the dogfood repro on
  mir3626/osint-stock-screener) converge to the upstream file on their next
  `vibe:sync` without conflict.
- No migration is required.
