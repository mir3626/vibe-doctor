# v1.5.8

## Summary

Adds provider-neutral upstream bootstrap for cloned harness projects.

## Changes

- `vibe-version-check` now best-effort infers `.vibe/config.json.upstream.url` from `git remote get-url origin` before deciding whether version checks are configured.
- `/vibe-init` invokes the same upstream bootstrap path without failing init when Git remotes are unavailable.
- Existing upstream configuration is preserved unchanged.
- Template self-sync is guarded so a `vibe-doctor` source checkout does not clone and sync itself unless `--from` is supplied.
- Added regression coverage for session-start bootstrap, `/vibe-init`, no-origin skip, existing-upstream preservation, and self-sync skip.
