# v1.5.2

## Summary

Workspace encoding hardening for Korean Markdown on Windows.

## Changes

- Added `.vscode/settings.json` to force UTF-8, LF line endings, and disable encoding auto-guessing.
- Added `.vscode/extensions.json` recommending EditorConfig support.
- Added sync-manifest coverage so downstream projects receive the editor encoding defaults.

## Notes

The repository already uses UTF-8 via `.editorconfig`. This patch covers editors that do not read EditorConfig early enough or occasionally mis-detect BOMless UTF-8 Markdown as a Windows legacy code page.
