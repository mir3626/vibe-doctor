# v1.7.25 - Hook Kill-Switch and Legacy-Model Override Shard

Date: 2026-07-06

## Changes

- Added a harness-wide hook kill-switch: setting `VIBE_HARNESS_HOOKS=off`
  (also `0`/`false`) in a session's environment makes every harness hook
  entry point — Stop QA gate, SessionStart, Notification, PostToolUse
  config-audit, and the PreCompact checkpoint — exit 0 immediately before
  doing any work. Background: downstream projects spawn harness-unrelated
  headless agents from their own product code (e.g., an API/CLI shelling out
  to a coding agent in the same repo), and those sessions inherited the
  repo's hooks, running `vibe:qa` on every Stop. The spawning code now sets
  one env var to opt the whole session out.
- Split the model-compensation rules out of the CLAUDE.md
  `HARNESS:mechanical-overrides` section into a new
  `docs/context/legacy-model-overrides.md` shard (staged execution with
  per-step approval, mandatory sub-agent swarming, re-read-before-edit,
  chunked file reads, truncation awareness, edit-integrity re-reads,
  six-way grep). SOTA-tier models (Claude Opus 4.5+/Fable-class,
  GPT-5.5-class and up) are instructed not to read the shard; only
  sub-SOTA Orchestrator/Generator sessions load and apply it. The three
  tier-independent rules (STEP 0 dead-code sweep, senior-developer
  override, mandatory verification) stay in CLAUDE.md.

## Compatibility

- Without `VIBE_HARNESS_HOOKS` set (or with any other value) every hook
  behaves byte-for-byte as in v1.7.24. The switch only short-circuits hook
  entry scripts; it never returns a non-zero exit, so it cannot block the
  host agent.
- The opted-out PreCompact path also skips the handoff auto-refresh — a
  headless product agent must not rewrite `.vibe/agent/handoff.md`.
- The new shard ships in the sync manifest `harness` list, so downstream
  projects receive it on `vibe:sync`; the slimmed `mechanical-overrides`
  section arrives through the existing CLAUDE.md section-merge. Downstream
  `PROJECT:custom-rules` content is untouched.
