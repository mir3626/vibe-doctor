# v1.5.10

## Summary

Fixes `/vibe-sync` staying pinned to an older `upstream.ref` after version-check detects a newer harness tag.

## Changes

- `/vibe-sync` now refreshes `.vibe/sync-cache.json` best-effort before resolving the upstream ref.
- Default sync uses cached `latestVersion` when it is newer than `harnessVersionInstalled`.
- Existing semver tag refs such as `v1.4.3` no longer trap downstream projects on the installed version.
- Explicit `--ref` remains the highest priority.
- Non-version refs such as `main` or feature branches remain preserved.
- Added regression coverage for stale pinned ref, explicit override, non-version ref, and stale cache behavior.

## Verification

- `npm run typecheck`
- `npm run build`
- `npm test`
