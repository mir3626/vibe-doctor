# v1.7.7 - Interview Consensus Gate

Date: 2026-05-06

LTS: `v1.7.3-lts` remains the stable baseline. This is a mainline harness behavior release for `/vibe-interview`.

## Changes

- Added a `phase: "consensus"` gate before `/vibe-interview` returns final `seedForProductMd`.
- Added `--consensus --decision approve|revise|defer|proxy-unconfirmed` to record the final human/proxy outcome.
- Kept active interview sessions open while consensus is pending or revised.
- Added a `Phase 3 Consensus Check` block to the final native interview seed, including status, summary hash, unresolved dimensions, and user corrections.
- Updated `/vibe-init` and `/vibe-interview` runbooks so PO-proxy finalization records `proxy-unconfirmed` instead of pretending human approval happened.
- Corrected the soft-termination documentation to match the implementation: required dimensions must reach coverage `>= 0.8` with ambiguity `<= 0.3`.

## Non-goals

- No changes to question selection, dimension weights, or ambiguity thresholds.
- No automatic semantic rewrite of coverage from consensus corrections; corrections are recorded explicitly and take precedence when they conflict with the dimension summary.
- No new persisted schema migration; older interview logs without `consensus` remain readable as historical artifacts.
