# v1.5.3

## Summary

WSL-safe Codex wrapper stdin and locale handling.

## Changes

- Prevented `chcp.com` from consuming piped prompt stdin by running it with `/dev/null` as stdin.
- Replaced the fixed `en_US.UTF-8` locale with an installed UTF-8 locale resolver that prefers `C.UTF-8` when available.
- Updated the Codex wrapper tests to stub `chcp.com` as a stdin-consuming command, catching regressions without a real authenticated Codex install.
- Sanitized inherited locale variables in wrapper tests so WSL images without `en_US.UTF-8` do not produce unrelated locale warnings.

## Notes

This keeps the provider-neutral lifecycle hooks from v1.5.1 and the UTF-8 Markdown/editor hardening from v1.5.2 intact. Downstream projects can sync this patch before re-running WSL `npm test`.
